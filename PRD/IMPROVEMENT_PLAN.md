### 서버

| 작업 | 상세 | 파일 |
|------|------|------|
| DB 마이그레이션 | `coordinator_matches` 테이블 추가, `consultation_requests`에 `coordinator_id`, `coordinator_note`, `matched_at` 컬럼 추가 | `migrations/002_coordinator.up.sql` |

## 6. ⚪ P3: 보안/품질 개선

### 보안 이슈

| 이슈 | 심각도 | 상세 | 파일 |
|------|--------|------|------|
| Admin API 인증 없음 | **Critical** | 관리자 엔드포인트에 미들웨어 미적용 — 누구나 접근 가능 | `routes.go` |
| 개발용 자격증명 하드코딩 | High | JWT 시크릿, Centrifugo 키가 "dev-secret" | `config.go`, `centrifugo.json` |
| CORS 전체 허용 | Medium | `allowed_origins: ["*"]` — 프로덕션에서는 도메인 제한 필요 | `cors.go`, `centrifugo.json` |
| 금액 키워드 필터링 | Medium | 현재 필터 파일(`pkg/filter/`) 존재하나 적용 범위 확인 필요 | `filter/` |

### 코드 품질

| 이슈 | 상세 |
|------|------|
| 에러 응답 표준화 | 일부 핸들러에서 에러 포맷 불일치 |
| Request 검증 강화 | `binding:"required"` 외 상세 검증 부족 (글자 수, 패턴 등) |
| N+1 쿼리 | repository 레이어에서 JOIN 최적화 필요 |
| 테스트 부재 | 유닛/통합 테스트 없음 |

### 인프라

| 이슈 | 상세 |
|------|------|
| Dockerfile 없음 | 서버 Dockerfile 미존재 — CI/CD 불가 |
| 이미지 저장소 | 로컬 파일시스템 → Cloudflare R2 전환 필요 (PRD 명세) |
| 프로덕션 환경 구성 | .env 프로덕션 값, SSL, 도메인 설정 필요 |

## 7. 기존 코드 중 유지/재사용 가능한 부분

| 영역 | 재사용 | 비고 |
|------|--------|------|
| 인증 (JWT + 카카오) | ✅ 거의 그대로 | 온보딩 텍스트만 변경 |
| 이미지 파이프라인 | ✅ 거의 그대로 | R2 전환 시 presigned URL 방식으로 변경 |
| 채팅 (Centrifugo) | ✅ 대부분 재사용 | 매칭 시 다중 채팅방 개설 로직 추가 |
| 커뮤니티 (게시물/댓글/좋아요) | ✅ 그대로 | 네이티브 광고 삽입 로직만 추가 (P2) |
| 투표/설문 | ✅ 그대로 | 변경 없음 |
| 병원 가입/승인 | ✅ 대부분 재사용 | 프리미엄 필드 추가 |
| 병원 탐색/검색 | ✅ 대부분 재사용 | 프리미엄 상단 노출 로직 추가 |
| 신고 시스템 | ✅ 그대로 | 변경 없음 |
| CMS 범용 기능 | ✅ 그대로 | 도메인 특화 메뉴 추가 |
