# 블랙라벨 — Feature Specifications

> 각 기능의 개요 + Acceptance Criteria 체크리스트.
> API 상세 스펙은 코드에서 관리. 이 문서에는 포함하지 않음.

---

## 1. Auth (인증)

### 1.1 일반 유저 가입

앱 실행 후 온보딩(3장 스와이프) -> 카카오 로그인 -> 이용약관 동의(필수 3 + 선택 2) -> 닉네임 설정 -> 관심 시술 선택(선택) -> 메인화면. JWT(access 15~30분 + refresh 14일) 기반 인증이며, refresh token은 Redis에서 관리하여 강제 로그아웃/탈퇴 시 즉시 무효화.

**AC**
- [ ] 카카오 로그인 성공 시 JWT 발급
- [ ] 필수 약관 3건(서비스, 개인정보, 만 18세) 미동의 시 가입 차단
- [ ] 닉네임 2~10자, 특수문자 제외, 중복 검사 통과
- [ ] 관심 시술 칩 멀티셀렉 + 스킵 가능
- [ ] 만 18세 미만 체크 시 가입 차단 + 안내 메시지

### 1.2 Apple 로그인 (MVP-B)

iOS 앱스토어 심사 필수 요건. iOS에서만 Apple Sign-in 버튼 노출. identity token 서버 검증 후 JWT 발급. 동일 이메일의 카카오 계정이 있으면 연동 안내.

**AC**
- [ ] iOS에서 Apple 로그인 버튼 노출, Android 미노출
- [ ] Apple identity token 서버 검증 정상 동작
- [ ] 신규 가입 시 약관 동의 플로우 동일 적용
- [ ] 기존 카카오 계정과 동일 이메일 시 연동 안내

### 1.3 계정 탈퇴

마이페이지 -> 탈퇴 사유 선택 -> "탈퇴합니다" 텍스트 직접 입력 -> 최종 확인. 7일 유예 기간(로그인 시 철회), 유예 후 개인정보 완전 삭제. 진행 중 상담 즉시 종료, 커뮤니티 게시물은 작성자명만 "탈퇴한 사용자"로 변환.

**AC**
- [ ] 탈퇴 신청 즉시 로그인 차단 + 프로필 비공개
- [ ] 7일 내 로그인 시 탈퇴 철회
- [ ] 7일 후 개인정보 + 사진 완전 삭제
- [ ] 진행 중 상담 즉시 종료, 병원에 알림 발송
- [ ] 재가입 시 이전 데이터 복구 불가

## 2. Consultation (코디네이터 매칭 상담)

유저가 시술 고민을 7단계 탭/칩 기반으로 등록하면, 전문 코디네이터가 요청을 검토한 뒤 맞춤 병원 2~3곳을 리스트업. 유저는 매칭된 모든 병원과 채팅 가능하며 직접 결정. 플랫폼은 연결까지만 담당. 배민 스타일 4단계 프로세스 UI로 대기 구간을 프리미엄 큐레이션 느낌으로 연출.

### 2.1 상담 접수 (7단계)

| Step | 내용 | UI |
|------|------|-----|
| 1 | 시술 카테고리 (성형외과/피부과) | 탭 단일 선택 |
| 2 | 세부 부위 | 칩 멀티셀렉 |
| 3 | 희망 시술 시기 | 탭 단일 선택 (1개월 이내 / 1~3개월 / 3~6개월 / 미정) |
| 4 | 사진 첨부 (필수, 1~5장) | 갤러리/카메라 |
| 5 | 상담 내용 (선택, 500자) | 자유 텍스트 |
| 6 | 민감정보 수집 동의 | 필수 체크 |
| 7 | 최종 확인 및 제출 | 요약 카드 |

**AC**
- [ ] 7단계 순차 입력 완료 후 제출 가능
- [ ] 사진 미첨부 시 제출 불가 (민감정보 미동의 시 텍스트 전용)
- [ ] 제출 후 배민 스타일 4단계 프로세스 UI 노출
- [ ] 코디네이터 SLA: 영업시간 내 2시간 이내 매칭
- [ ] 매칭 완료 시 유저/병원 양측 푸시 알림
- [ ] 매칭된 병원 2~3개 카드 표시, 모든 병원과 채팅 가능
- [ ] 금액 키워드 필터링 ("만원", "원", 숫자+단위 등) 작동

### 2.2 상담 임시저장 (MVP-B, P2)

Step 이동 시 로컬 DB(SQLite/drift)에 자동 저장. 앱 재진입 시 "이어서 작성할까요?" 다이얼로그. 7일 후 자동 만료.

**AC**
- [ ] Step 이동 시 로컬 자동 저장
- [ ] 앱 재진입 시 이어하기 다이얼로그 표시
- [ ] "새로 작성" 선택 시 임시저장 삭제
- [ ] 7일 경과 시 자동 만료
- [ ] 제출 성공 시 임시저장 클리어

### 2.3 코디네이터 매칭 (CMS)

CMS에서 코디네이터가 새 상담 요청을 확인하고 파트너 병원 풀에서 시술 전문성/응답률/분배 균형 기준으로 2~3곳 선정. 매칭 확정 시 채팅방 자동 개설. 프리미엄 병원은 정보가 더 풍부할 뿐 매칭 우선권은 없음.

**AC**
- [ ] CMS에서 상담 요청 큐 확인 가능
- [ ] 코디네이터가 2~3개 병원 매칭 + 사유 메모
- [ ] 매칭 확정 시 채팅방 자동 개설 (2~3개)
- [ ] 영업시간 내 2시간 초과 미매칭 시 에스컬레이션 알림

## 3. Chat (1:1 채팅)

Centrifugo + WebSocket 기반 실시간 채팅. 코디네이터 매칭 완료 시 요청 1건당 2~3개 채팅방 자동 개설. 텍스트(1,000자), 사진(5장/회), 시스템 메시지, 방문 예약 카드 지원. 면책 고지 배너 상단 상시 고정. 종료 조건: 유저 직접 종료 / 상담 완료 / 30일 비활성 자동 종료.

**AC**
- [ ] 코디네이터 매칭 시 채팅방 자동 개설 (복수)
- [ ] 면책 고지 배너 채팅방 상단 항상 표시
- [ ] 읽음 확인 (상대방 읽은 시각 노출)
- [ ] 금액/연락처/카카오ID 키워드 필터링 작동
- [ ] WebSocket 끊김 시 자동 재연결 (최대 5회, 지수 백오프)
- [ ] 30일 비활성 시 자동 종료 + 읽기 전용 전환
- [ ] 오프라인 -> 온라인 복귀 시 미수신 메시지 자동 동기화

### 3.1 방문 예약 카드 (MVP-B)

병원만 발송 가능. 병원명 + 주소 + 제안 날짜/시간 + "수락" / "다른 날짜 요청" 버튼. 수락 시 양측 확정 알림. 구속력 없음.

**AC**
- [ ] 병원만 방문 예약 카드 발송 가능
- [ ] 카드에 병원 주소, 제안 날짜/시간 표시
- [ ] 유저 "수락" -> 양측에 확정 알림
- [ ] 유저 "다른 날짜" -> 텍스트 메시지로 전환
- [ ] 수락된 카드는 변경 불가 (새 카드로 재발송만 가능)

### 3.2 키워드 필터 정규식

서버/클라이언트 이중 필터링. 발송 차단 + 발신자에게 안내 메시지 표시.

| 패턴 | 대상 |
|------|------|
| `\d{1,4}\s*(만\s*원\|원\|만)` | 채팅, 상담, 게시글, 댓글, 리뷰 |
| `01[016789]-?\d{3,4}-?\d{4}` | 채팅, 게시글, 댓글, 리뷰 |
| `(카[카톡]\|kakao\|open\.kakao)\s*:?\s*\S+` | 채팅, 게시글, 댓글, 리뷰 |
| `[일이삼사오육칠팔구십백천]\s*(만\s*원\|원)` | 채팅, 상담 |

## 4. Community (커뮤니티)

### 4.1 비포&애프터 게시판 (MVP-A)

유저 자율 커뮤니티. 플랫폼 인센티브 없음 (의료광고 간접 유도 리스크 해소). 시술 카테고리 태그(필수), 전후 사진(선택, 최대 10장), 후기 텍스트(최소 20자), 병원 태깅(선택). 댓글/대댓글/좋아요 지원.

**AC**
- [ ] 시술 카테고리 태그 필수, 텍스트 최소 20자
- [ ] 병원 태깅 시 탭하면 병원 상세로 이동
- [ ] 병원명/의사명/타 플랫폼 링크 포함 시 게시 차단
- [ ] 금액 직접 언급 시 `***` 블라인드 처리
- [ ] 신고 기능 -> CMS 검수 후 블라인드

### 4.2 Before/After 슬라이더 (MVP-B)

비포&애프터 게시글에서 전후 사진을 좌우 드래그 슬라이더로 인터랙티브 비교.

**AC**
- [ ] 게시글 작성 시 사진에 Before/After 태그 지정 가능
- [ ] 상세 화면에서 슬라이더 드래그로 전후 비교
- [ ] 슬라이더 <-> 일반 갤러리 뷰 토글
- [ ] 민감정보 미동의 시 블러 처리
- [ ] Before/After 태그 없는 사진은 일반 갤러리로 표시

### 4.3 자유 게시판 / Q&A (MVP-B)

성형 관련 자유 의견 교환. 익명 게시 지원. 태그 기반 분류 (#눈성형, #코성형, #회복기간 등). 댓글/대댓글/좋아요 지원.

### 4.4 투표 / 설문 (MVP-B)

유저 또는 운영팀이 투표 생성. 이미지 포함 투표, 실시간 결과 공개(참여 후), 인기 투표 메인 피드 노출.

### 4.5 네이티브 광고 (3차)

커뮤니티 피드에 콘텐츠형 광고 카드 삽입. 일반 게시물 5개당 1개, 연속 금지, "광고" 라벨 필수. 좋아요/댓글 없음. 충전형 CPM 과금. CMS에서 소재 사전 승인 후 게재.

**AC**
- [ ] 일반 게시물 5개당 광고 1개 삽입
- [ ] 광고끼리 연속 노출 안 됨
- [ ] "광고" 라벨 필수 표시
- [ ] 카드 탭 시 병원 상세로 이동
- [ ] 의료법 56조 준수 확인 후 게재

### 4.6 콘텐츠 운영 정책

| 구분 | 규칙 |
|------|------|
| 자동 차단 (게시 불가) | 병원명, 의사 이름, 타 플랫폼 링크 |
| 자동 블라인드 `***` | 욕설/비방, 가격 정보 직접 언급 |
| 관리자 승인 후 게시 | 극단적 신체 노출 사진 |

## 5. Hospital (병원)

### 5.1 병원 가입 승인

사업자등록번호 입력(국세청 API 검증) -> 의료기관 개설 신고증 업로드 -> 기본 정보 입력 -> CMS 수동 승인(1~3영업일). 반려 시 사유 안내 + 재신청 가능.

**AC**
- [ ] 사업자등록번호 국세청 API 유효성 검증
- [ ] CMS에서 신고증 확인 후 승인/거부
- [ ] 승인 완료 시 병원 계정 활성화 + 알림

### 5.2 병원 탐색

리스트 뷰(MVP-A): 병원명/시술 분야 검색 + 필터(시술/지역) + 정렬(일치순/거리/후기). 지도 뷰(MVP-B): 리스트 <-> 지도 전환, 마커 탭 시 병원 카드 미리보기. 프리미엄 병원 검색 상단 노출(MVP-B): 최대 3개, "광고" 라벨 필수.

**AC**
- [ ] 병원명/시술 분야 키워드 검색 동작
- [ ] 필터(시술 분야, 지역) 정상 동작
- [ ] 프리미엄 병원 상단 최대 3개 + "광고" 라벨 표시

### 5.3 병원 상세 페이지

무료: 기본 정보 + 전문 시술 칩 5개 + 소개 300자 + 사진 3장 + 의료진 1명 + 지도 + 커뮤니티 태깅 후기 + 출연자 후기 + 리뷰 + "상담 요청하기" CTA.
프리미엄(MVP-B): 칩 10개 + 소개 1,000자 + 사진 10장 + 소개 영상 1개 + 전체 의료진 + 시술 상세.

## 6. Cast Member (출연자)

렛미인 유튜브 출연 경험자를 인증된 출연자(Cast Member)로 관리. 인증 배지를 통해 신뢰도 확보하고, 스토리 피드를 통해 팬덤을 플랫폼으로 락인.

### 6.1 출연자 인증 (MVP-A)

마이페이지에서 "출연자 인증 신청" -> 출연 회차 + 본인 확인 자료 제출 -> CMS 관리자 검토(유튜브 영상 대조) -> 승인 -> "렛미인 출연자" 인증 배지 + 스토리 업로드 권한 활성화.

**AC**
- [ ] 출연자 인증 신청 UI (마이페이지)
- [ ] CMS에서 유튜브 영상 대조 확인 후 승인/반려
- [ ] 승인 시 닉네임 옆 인증 배지 노출 (게시물, 댓글, 스토리, 채팅)
- [ ] 배지 박탈 가능 (CMS 관리자)

### 6.2 스토리 피드 (MVP-B)

출연자만 업로드 가능. 텍스트(10~1,000자) + 사진(최대 10장) + 유튜브 URL + 출연 에피소드 자동 고정. 회복 일지 / Q&A / 일상 유형. 유저 질문 댓글에 "출연자 답변" 배지 + 상단 고정.

**AC**
- [ ] 출연자만 스토리 포스트 업로드 가능
- [ ] 스토리 하단에 출연 에피소드 링크 자동 고정
- [ ] 출연자 답변에 배지 표시 + 댓글 상단 고정 옵션

### 6.3 팔로우

출연자 프로필/스토리에서 "팔로우" 버튼. 팔로우 시 홈 피드 상단 노출 + 새 스토리 업로드 시 푸시 알림. 마이페이지에서 팔로잉 목록 확인.

**AC**
- [ ] 팔로우/언팔로우 정상 동작
- [ ] 팔로우 중인 출연자 스토리가 홈 피드 상단 노출
- [ ] 새 스토리 업로드 시 팔로워에게 푸시 알림

### 6.4 홈 스토리 바

홈 상단에 인스타그램 스타일 원형 프로필 가로 스크롤. 새 스토리 있는 출연자는 레드/버건디 테두리. 팔로우 중인 출연자 우선 정렬. 탭 시 해당 출연자 최신 스토리로 이동.

### 6.5 유튜브 에피소드 관리

CMS에서 에피소드 등록 (유튜브 URL, 제목, 출연자 연결, 방영일). 썸네일 자동 수집. 선택된 에피소드는 홈 히어로 캐러셀에 노출.

## 7. Review (병원 리뷰, MVP-B)

병원 상담을 실제 진행한 유저만 작성 가능한 검증된 리뷰 시스템. 코디네이터 매칭을 통해 CHATTING 또는 COMPLETED 상태인 유저에게만 작성 권한 부여. 별점(1~5, 0.5단위) + 텍스트(30~1,000자) + 사진(최대 5장, 선택). 병원-매칭 조합 기준 1건만 허용(수정 가능).

### 7.1 작성 자격

| 매칭 상태 | 리뷰 작성 |
|----------|----------|
| PENDING / MATCHED | 불가 |
| CHATTING / COMPLETED | 가능 |
| CANCELLED | 불가 |

**AC**
- [ ] CHATTING/COMPLETED 아닌 유저는 작성 버튼 비활성
- [ ] 자격 없으면 "상담 진행 후 리뷰 작성 가능" 안내
- [ ] 매칭 건별로 각각 1건 리뷰만 허용

### 7.2 리뷰 작성/수정/삭제

**AC**
- [ ] 별점 미선택 시 제출 불가
- [ ] 텍스트 30자 미만 / 1,000자 초과 시 에러
- [ ] 금액/연락처 키워드 포함 시 제출 차단 + 구체적 사유 안내
- [ ] 사진 5장 초과 / 10MB 초과 시 안내
- [ ] 동일 (hospital_id, match_id) 중복 작성 시 409 Conflict
- [ ] 수정된 리뷰에 "(수정됨)" 라벨 + 최종 수정 시각
- [ ] 삭제 시 확인 다이얼로그, 삭제 후 평균 별점 즉시 재계산

### 7.3 병원 상세 노출

평균 별점(소수점 1자리) + 별점 분포 막대 그래프 + 총 리뷰 수. 정렬(최신순/별점순). 커서 기반 무한 스크롤. 본인 리뷰 상단 고정 + 수정/삭제 버튼.

**AC**
- [ ] 평균 별점 + 리뷰 수 표시
- [ ] 리뷰 0건 빈 상태 UI 정상 표시
- [ ] 정렬 변경 시 즉시 갱신
- [ ] 무한 스크롤 페이지네이션
- [ ] 본인 리뷰 최상단 고정 + 수정/삭제 버튼 노출

## 10. Hospital Portal (병원 전용 기능)

### 10.1 병원 대시보드

매칭 수신 건 / 채팅 활성 건 / 상담 완료 건 현황 확인.

### 10.2 프로필 관리

병원 소개, 전문 분야, 위치, 사진, 의료진 정보 관리. 프리미엄 구독 시 정보 확장 가능.

### 10.3 상담 관리

코디네이터 매칭 시 푸시 알림 수신. 매칭된 유저 요청 상세(시술/사진/텍스트) 확인. 채팅 참여.

### 10.4 광고 관리 (3차)

광고 크레딧 충전(최소 5만원) + 캠페인 생성(지면/기간/일예산/소재) + 성과 리포트(노출/클릭/CTR). 크레딧 소진 시 자동 중단.

### 10.5 프리미엄 구독 (MVP-B)

월정액 구독으로 프로필 정보 확장 + 검색 상단 노출 + 홈 추천 영역 노출. 매칭 기능과 완전 분리 (무료 병원도 동등한 매칭 참여).

**AC**
- [ ] 프리미엄/무료 병원 모두 매칭 참여 동일 조건
- [ ] 프리미엄 병원은 정보량만 확장 (매칭 우선권 없음)
- [ ] 프리미엄 상단 노출 시 "광고" 라벨 필수

## 11. Media (이미지 처리)

모든 이미지(상담 사진, 비포&애프터, 병원 프로필, 채팅, 리뷰)에 적용되는 공통 파이프라인. Cloudflare R2 + CDN 기반.

### 11.1 업로드 처리

클라이언트: EXIF 제거 + 사전 압축(최대 2048px) -> presigned URL 업로드. 서버 후처리(비동기): 리사이징 3종(썸네일 300px + 중간 800px + 풀 2048px) + EXIF 완전 제거 + WebP 변환.

### 11.2 저장 구조

| 영역 | 접근 권한 |
|------|----------|
| 상담 요청 사진, 채팅 사진 | 비공개 버킷 (해당 유저 + 매칭 병원만) |
| 비포&애프터 사진, 병원 프로필, 리뷰 사진 | 공개 버킷 (CDN) |

### 11.3 용량 제한

| 항목 | 제한 |
|------|------|
| 단일 이미지 | 최대 10MB |
| 상담 요청 / 채팅 1회 / 리뷰 | 최대 5장 |
| 커뮤니티 게시물 | 최대 10장 |

### 11.4 삭제 정책

유저 탈퇴(7일 후 완전 삭제), 민감정보 동의 철회(즉시 삭제), 채팅 기록 삭제(사진 함께), 게시물 삭제/블라인드(사진 삭제).

**AC**
- [ ] Presigned URL 업로드 정상 동작
- [ ] 후처리로 3종 리사이즈 + WebP 변환 완료
- [ ] EXIF(GPS 포함) 완전 제거
- [ ] 비공개/공개 버킷 접근 제어 정상
- [ ] 탈퇴/동의철회 시 관련 사진 완전 삭제
