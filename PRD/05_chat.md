## 채팅 종료 조건 — 3가지 모두 적용

| 조건 | 내용 |
|------|------|
| 유저 직접 종료 | "상담 종료" 버튼 → 확인 다이얼로그 → 종료 |
| 상담 완료 처리 | 유저 또는 병원이 "상담 완료" → 양측 확인 후 종료 |
| 자동 종료 | 마지막 메시지 기준 30일 비활성 시 자동 종료 (운영 정책으로 조정 가능) |

종료 후 채팅방은 읽기 전용 전환 (재개설 불가, 기록만 열람)

### 키워드 필터 정규식 *(v2.2 추가)*

> 서버/클라이언트 이중 필터링. 발송 차단 + 발신자에게 안내 메시지 표시.

| 패턴 | 정규식 | 대상 |
|------|--------|------|
| 금액 | `\d{1,4}\s*(만\s*원\|원\|만)` | 채팅, 상담 메모, 게시글, 댓글 |
| 전화번호 | `01[016789]-?\d{3,4}-?\d{4}` | 채팅, 게시글, 댓글 |
| 카카오 ID | `(카[카톡]\|kakao\|open\.kakao)\s*:?\s*\S+` | 채팅, 게시글, 댓글 |
| 한글 숫자 우회 | `[일이삼사오육칠팔구십백천]\s*(만\s*원\|원)` | 채팅, 상담 메모 |

```json
// 422 응답 예시
{ "error": "금지된 표현이 포함되어 있습니다", "code": "KEYWORD_BLOCKED" }
```

## 채팅에서 방문 전환

> 외부 연락처를 차단하면서 실제 방문으로 연결하는 경로 설계.

| 항목 | 내용 |
|------|------|
| 방문 예약 안내 | 병원이 "방문 상담 안내" 시스템 메시지를 발송 가능 (병원 주소·진료시간 자동 포함) → [04_hospital.md](./04_hospital.md) |
| 방문 확인 | 유저가 "방문 예정" 버튼 탭 → 날짜·시간 선택 → 양측 확인 (단순 일정 공유, 구속력 없음) |
| 외부 연락처 전환 | 양측 합의 후 유저가 "연락처 공유" 동의 시에만 전화번호 교환 가능 |

### 방문 예약 카드 상세 *(v2.2 추가)*

> 병원이 채팅에서 방문 일시를 제안하고, 유저가 수락/거절하는 인터랙티브 카드.

| 항목 | 내용 |
|------|------|
| 발송 주체 | 병원만 가능 (유저는 텍스트로 희망 일시 전달) |
| 카드 구성 | 병원명 + 주소 + 제안 날짜/시간 + "수락" / "다른 날짜 요청" 버튼 |
| 수락 시 | 양측에 "방문 예약 확정" 시스템 메시지 + 푸시 알림 → [15_notification.md](./15_notification.md) |
| 다른 날짜 요청 | 유저가 텍스트로 희망 일시 전달 → 병원이 새 카드 재발송 |
| 구속력 | 없음 — 단순 일정 공유 목적 |

**API**
```json
// POST /api/v1/chat/rooms/:id/visit-card (병원 전용)
// Req:  { "proposedDate": "2026-04-05", "proposedTime": "14:00", "note": "2층 상담실로 오세요" }
// Res:  { "id": 1, "roomId": 3, "status": "proposed", "proposedDate": "2026-04-05", "proposedTime": "14:00" }

// POST /api/v1/chat/rooms/:id/visit-card/:cardId/accept (유저)
// Res:  { "id": 1, "status": "accepted" }

// POST /api/v1/chat/rooms/:id/visit-card/:cardId/decline (유저)
// Req:  { "message": "오후 3시는 가능할까요?" }
// Res:  { "id": 1, "status": "declined" }
```

**AC**
- [ ] 병원만 방문 예약 카드 발송 가능
- [ ] 카드에 병원 주소, 제안 날짜/시간 표시
- [ ] 유저 "수락" → 양측에 확정 알림
- [ ] 유저 "다른 날짜" → 텍스트 메시지로 전환
- [ ] 수락된 카드는 변경 불가 (새 카드로 재발송만 가능)

## 네트워크 오류 처리

| 상황 | 처리 |
|------|------|
| WebSocket 연결 끊김 | 자동 재연결 시도 (최대 5회, 지수 백오프). 실패 시 "연결이 불안정합니다" 배너 표시 |
| 메시지 전송 실패 | 메시지 옆 "재전송" 버튼 표시. 미전송 메시지 로컬 큐에 보관 |
| 사진 전송 실패 | "재시도" 버튼 + 진행률 표시. 백그라운드 전환 시에도 전송 지속 |
| 오프라인 → 온라인 복귀 | 미수신 메시지 자동 동기화 (Centrifugo history 기능 활용) |
