## 면책 고지

> ⚠️ 채팅 내 의료 상담 시 비대면 진료 규제 위반 가능성 방지. → [10_legal.md](./10_legal.md)

| 항목 | 내용 |
|------|------|
| 노출 시점 | 채팅방 개설 시 시스템 메시지로 자동 표시 |
| 고지 내용 | "이 채팅은 의료 진단·처방이 아닌 상담 안내 목적입니다. 정확한 진단은 대면 진료를 통해 받으시기 바랍니다." |
| 고정 여부 | 채팅방 상단에 항상 고정 배너로 표시 |

## 채팅 종료 조건 — 3가지 모두 적용

| 조건 | 내용 |
|------|------|
| 유저 직접 종료 | "상담 종료" 버튼 → 확인 다이얼로그 → 종료 |
| 상담 완료 처리 | 유저 또는 병원이 "상담 완료" → 양측 확인 후 종료 |
| 자동 종료 | 마지막 메시지 기준 30일 비활성 시 자동 종료 (운영 정책으로 조정 가능) |

종료 후 채팅방은 읽기 전용 전환 (재개설 불가, 기록만 열람)

## 채팅에서 방문 전환

> 외부 연락처를 차단하면서 실제 방문으로 연결하는 경로 설계.

| 항목 | 내용 |
|------|------|
| 방문 예약 안내 | 병원이 "방문 상담 안내" 시스템 메시지를 발송 가능 (병원 주소·진료시간 자동 포함) → [04_hospital.md](./04_hospital.md) |
| 방문 확인 | 유저가 "방문 예정" 버튼 탭 → 날짜·시간 선택 → 양측 확인 (단순 일정 공유, 구속력 없음) |
| 외부 연락처 전환 | 양측 합의 후 유저가 "연락처 공유" 동의 시에만 전화번호 교환 가능 |

## 푸시 알림 이벤트

| 이벤트 | 알림 내용 |
|--------|----------|
| 새 메시지 수신 | "○○병원에서 메시지가 왔습니다" |
| 채팅 개설 | "○○병원과 상담이 시작됐습니다" |
| 자동 종료 7일 전 | "상담이 7일 후 자동 종료됩니다" |
| 채팅 종료 | "○○병원과의 상담이 종료됐습니다" |
| 기록 삭제 N일 전 | "채팅 기록이 N일 후 삭제됩니다" |

## 네트워크 오류 처리

| 상황 | 처리 |
|------|------|
| WebSocket 연결 끊김 | 자동 재연결 시도 (최대 5회, 지수 백오프). 실패 시 "연결이 불안정합니다" 배너 표시 |
| 메시지 전송 실패 | 메시지 옆 "재전송" 버튼 표시. 미전송 메시지 로컬 큐에 보관 |
| 사진 전송 실패 | "재시도" 버튼 + 진행률 표시. 백그라운드 전환 시에도 전송 지속 |
| 오프라인 → 온라인 복귀 | 미수신 메시지 자동 동기화 (Centrifugo history 기능 활용) |
