## 1. 서비스 개요

### 1.1 서비스 비전

성형을 고민하는 사람들이 정보의 불균형 없이 신뢰할 수 있는 커뮤니티 안에서 실제 경험을 공유하고, 필요 시 병원과 자연스럽게 연결되는 플랫폼.

### 1.2 핵심 문제 정의

- 성형 시장의 정보 불균형 — 실제 후기보다 병원 마케팅 콘텐츠에 과다 노출
- 커뮤니티 파편화 — 카페·카톡·인스타 등 정보 채널 분산
- 비포&애프터 정보 부족 — 병원 직접 게시 불가(의료광고 규제)로 인한 정보 공백
- 병원 연결 과정의 불투명성 — 가격·시술 정보 비교 어려움

### 1.3 목표 사용자

| 구분 | 특성 | 핵심 니즈 |
|------|------|----------|
| 일반 유저 (소비자) | 성형 고민 중인 20–40대 여성 중심 | 신뢰할 수 있는 실제 후기, 병원 정보, 커뮤니티 |
| 병원 (B2B) | 성형외과 원장/마케터 | 잠재 고객 확보, 정보 노출 |
| CMS 관리자 | 플랫폼 운영팀 | 콘텐츠 검수, 유저/병원 관리, 신고 처리 |

### 1.4 플랫폼 포지셔닝

**성형 정보·커뮤니티 플랫폼**으로 포지셔닝한다.
역경매 상담 연결 기능은 포함되어 있으나, 수익 모델은 추후 법률 검토 후 별도 확정한다.
커뮤니티와 콘텐츠가 서비스의 핵심 가치이며, 병원 연결은 부가 기능으로 위치한다.

### 1.5 경쟁 환경 및 차별점 *(v1.4 신규)*

| 기존 플랫폼 | 특징 | 우리의 차별점 |
|------------|------|-------------|
| 강남언니 | 병원 리스팅 + 리뷰 + 광고 중심 | 역경매 구조로 유저 주도권 확보 |
| 바비톡 | 커뮤니티 + 병원 정보 | 사진 기반 역경매로 정밀 상담 연결 |
| 여신티켓 | 할인·쿠폰 중심 | 커뮤니티 + 역경매 복합 모델 |

> 핵심 차별점: "유저가 사진과 고민을 올리면 병원이 찾아오는 역경매 구조"
> 기존 플랫폼은 유저가 병원을 탐색하는 구조이지만, 우리는 병원이 유저에게 먼저 다가오는 구조.

## 2. 역경매 상담 요청 시스템

> ⚠️ 기능은 포함하되, **수익 모델(병원 과금 구조)은 법률 검토 후 별도 확정**. 현재는 무과금 구조로 개발 진행.
> 의료법 제27조 관련 법률 자문은 출시 전 필수.

### 2.1 구조 개요

유저가 시술 고민을 등록하면 병원들이 열람 후 상담 메시지를 발송하는 역경매 구조.
1개 요청당 최대 **5개 병원** 선착순 발송. 유저는 수신된 상담을 비교 후 1개 병원을 선택해 1:1 채팅으로 전환한다.

---

### 2.2 유저 상담 요청 플로우

#### Step 1 — 시술 카테고리 선택

| 항목 | 내용 |
|------|------|
| UI | 아이콘 + 텍스트 그리드 (2열) |
| 선택 방식 | 단일 선택 |
| 카테고리 | 눈 / 코 / 윤곽 / 가슴 / 지방흡입 / 피부·레이저 / 기타 |
| 관리 | CMS에서 추가·수정·삭제 |

#### Step 2 — 세부 시술 선택

| 항목 | 내용 |
|------|------|
| UI | 칩(Chip) 멀티셀렉 |
| 선택 방식 | 복수 선택 가능 |
| 예시 (눈) | 쌍꺼풀 매몰법 / 절개법 / 앞트임 / 뒤트임 / 눈밑지방 / 안검하수 교정 |
| 예시 (코) | 콧대 / 코끝 / 들창코 / 매부리코 / 비중격 교정 |
| 관리 | CMS에서 카테고리별 항목 관리 |

#### Step 3 — 사진 촬영 및 업로드

**기본 원칙**
- 보정·필터 완전 차단 — 카메라 UI 필터 옵션 미노출, 원본 업로드
- 앱 내 카메라 실행 (갤러리 업로드 병행 허용, 단 가이드 프레임 미제공)
- 업로드 전 안내: "보정 없는 사진이 정확한 상담에 도움됩니다"
- 촬영 사진은 병원에게만 공개 (커뮤니티 비공개)

**부위별 촬영 가이드 프레임**

앱 내 카메라 실행 시 반투명 실루엣 오버레이 표시. 각도별로 순서대로 촬영.

| 부위 | 촬영 장수 | 각도 순서 | 프레임 형태 |
|------|----------|-----------|------------|
| 눈 | 3장 | ① 정면(눈 뜬) → ② 정면(눈 감은) → ③ 클로즈업(좌 또는 우) | 눈 영역 타원형 |
| 코 | 4장 | ① 정면 → ② 좌측면(90°) → ③ 우측면(90°) → ④ 45도 사선 | 코 중심 직사각형 |
| 윤곽 (턱·광대) | 4장 | ① 정면 → ② 좌측면 → ③ 우측면 → ④ 45도 사선 | 얼굴 전체 윤곽 |
| 가슴 | 3장 | ① 정면 → ② 좌측면 → ③ 우측면 | 흉부 영역 직사각형 |
| 지방흡입 | 3장 | ① 정면 → ② 측면 → ③ 후면 | 해당 부위 영역 |
| 기타 | 3장 | ① 클로즈업 정면 → ② 측면 → ③ 후면 | 자유 프레임 |

**촬영 UX**
- 각도별 한 장씩 순서 안내 (각도명 + 예시 일러스트)
- 완료된 장 하단 썸네일 표시, 재촬영 가능
- 모든 각도 완료 시 다음 Step 자동 이동
- 조명 부족 감지 시 경고 문구 표시

**사진 공개 토글**

| 설정 | 내용 |
|------|------|
| ON (기본값) | 병원이 사진 포함 전체 열람 가능 |
| OFF | 텍스트만 노출, 사진은 1:1 채팅 전환 후 공유. 안내 문구: "사진 없이는 병원의 상담 응답률이 낮아질 수 있습니다" |

#### Step 4 — 상담 요청 내용 입력

| 항목 | 내용 |
|------|------|
| UI | 자유 텍스트 멀티라인 |
| 글자 수 | 최소 20자 / 최대 500자 |
| 추가 항목 | 희망 시술 시기 (1개월 이내 / 3개월 이내 / 6개월 이내 / 미정) |

#### Step 5 — 공개 범위 (MVP)

- MVP: 전체 병원 공개 단일 옵션
- 안내: "등록된 병원들이 요청을 확인하고 상담을 보내드립니다"

#### Step 6 — 민감정보 수집 동의 *(v1.4 신규)*

> ⚠️ 신체 사진은 개인정보보호법상 민감정보에 해당. 일반 개인정보 동의와 **별도의 민감정보 수집·이용 동의**가 필수.

| 항목 | 내용 |
|------|------|
| 동의 항목 | "민감정보(신체 사진) 수집·이용 동의" (필수) |
| 동의 내용 | 수집 목적, 보관 기간, 제3자(병원) 제공 범위 명시 |
| 미동의 시 | 사진 업로드 불가, 텍스트 전용 요청만 가능 |
| 동의 철회 | 마이페이지에서 언제든 철회 가능 → 기존 사진 즉시 삭제 |

#### Step 7 — 최종 확인 및 제출

| 항목 | 내용 |
|------|------|
| UI | 요약 카드 (시술, 사진 썸네일, 요청 내용, 시기) |
| 제출 후 안내 | "72시간 내에 최대 5개 병원에서 상담이 도착할 예정입니다" |
| 요청 유효 기간 | 72시간 |
| 알림 | 상담 도착 시 푸시 알림 |
| 취소 | 72시간 내 취소 가능 (발송된 상담은 유지) |

### 2.4 병원 — 상담 메시지 작성

#### 구조화 필드 (필수)

| 필드명 | 입력 방식 | 설명 |
|--------|----------|------|
| 병원 한 줄 소개 | 텍스트 최대 60자 | 프로필 자동완성, 수정 가능 |
| 전문 시술 분야 | 칩 선택 최대 3개 | 프로필 등록 분야 기반 자동 노출 |
| 해당 시술 경력 | 텍스트 최대 60자 | 예: "쌍꺼풀 절개법 전문, 15년 경력" |
| 상담 가능 방식 | 체크박스 | 방문 / 비대면 화상 / 채팅 |
| 상담 가능 시간 | 텍스트 또는 시간 선택 | 예: "평일 10:00–18:00" |

#### 자유 텍스트 (필수)

- 최소 50자 / 최대 1,000자

#### 선택 항목

| 필드명 | 내용 |
|--------|------|
| 병원 사진 | 최대 3장 (병원 내부·장비만, 시술 전후 불가) |
| 첫 상담 안내 문구 | 최대 100자 |

#### 금액 기재 차단

- **키워드 필터링**: "만원", "원", 숫자+단위 패턴 감지 시 발송 차단 + 경고 문구 표시
- 필터링 우회 시도(한글 변환 등) 감지 로직 추가
- 위반 적발 시: 경고 1회 → 2회 누적 시 계정 정지

#### 발송 제약

| 항목 | 내용 |
|------|------|
| 발송 상한 | 요청당 최대 5개 병원 (선착순) |
| 발송 취소 | 유저 선택 전까지 취소 가능 |
| 수정 | 발송 후 수정 불가 |
| 재발송 | 동일 요청 재발송 불가 |
| 추가 금지 | 외부 연락처 유도 (전화번호, 카카오톡 ID 등) |

### 2.6 엣지 케이스 및 예외 처리 *(v1.4 신규)*

| 상황 | 처리 |
|------|------|
| 72시간 내 병원 응답 0건 | "아직 상담이 도착하지 않았습니다. 새로운 요청을 등록해 보세요" + 재요청 버튼 |
| 72시간 내 5슬롯 미충족 (예: 2건만 도착) | 유효 기간 만료 후 수신된 상담만으로 선택 가능. 추가 요청은 새 건으로 등록 |
| 재요청 | 동일 내용 재요청 가능, 쿨다운 없음. 단 이전 요청에 응답한 병원은 중복 노출 방지 |
| 유저가 병원 선택 후 나머지 병원 | 미선택 병원에 시스템 알림: "다른 병원이 선택되었습니다" (병원명 비공개) |
| 유저가 아무 병원도 선택 안 함 | 72시간 만료 후 요청 자동 종료. 병원에 "요청이 종료되었습니다" 알림 |
| 사진 업로드 실패 | 재시도 버튼 + "네트워크를 확인해 주세요" 안내. 임시 저장 후 이어서 업로드 지원 |

## 3. 성형 커뮤니티

### 3.1 비포&애프터 게시판

> ⚠️ 유저 자율 커뮤니티로 운영. 플랫폼은 인센티브를 제공하지 않으며, 자연 발생 콘텐츠 구조로 설계. (의료광고 간접 유도 리스크 해소)

**운영 원칙**
- 유저가 자발적으로 시술 전후 사진 + 후기 게시
- **플랫폼 인센티브 없음** — 포인트·리워드 미제공
- 병원 태깅 허용 (유저 자발적 선택, 강제 아님)
- 태깅된 병원은 해당 게시물 승인·거부 불가 (유저 주도권 유지)
- 신고 기능 → CMS 검수 후 블라인드

**게시물 구성**
- 시술 카테고리 태그 (필수)
- 시술 시기 입력 (선택)
- 전후 사진 (선택, 최대 10장)
- 후기 텍스트 (최소 20자)
- 병원 태깅 (선택)

**초기 콘텐츠 확보 전략** *(v1.4 신규)*

> 자율 커뮤니티 모델이므로 인센티브 없이 초기 콘텐츠를 확보하는 전략이 필요.

| 전략 | 내용 |
|------|------|
| 시드 콘텐츠 | 운영팀이 실제 시술 경험자 10~20명 섭외하여 초기 후기 게시 요청 (무보상, 순수 자발) |
| 커뮤니티 이전 유도 | 기존 성형 카페·커뮤니티 활동 유저 대상 앱 안내 |
| 유튜브 채널 연동 | 운영사 유튜브 시청자를 앱 커뮤니티로 유도 |
| Q&A 시드 | 운영팀이 자주 묻는 질문 20~30개 사전 등록 |

### 3.2 자유 게시판 / Q&A

- 성형 관련 자유 의견 교환
- 익명 게시 지원
- 태그 기반 분류 (#눈성형, #코성형, #회복기간 등)
- 댓글·대댓글·좋아요 지원

### 3.3 투표 / 설문

- 유저 또는 운영팀이 투표 생성
- 이미지 포함 투표 지원
- 결과 실시간 공개 (참여 후 노출)
- 설문: 복수 선택형 + 주관식 혼합
- 인기 투표 메인 피드 노출 → 커뮤니티 활성화 유도

## 5. 레퍼럴 이벤트

> ⚠️ 병원 방문·상담 완료를 트리거로 하는 레퍼럴은 의료법상 알선 대가로 해석될 수 있어 제외.
> 앱 가입 기반으로만 운영.

### 5.1 구조

- 유저 A가 친구 B에게 추천인 코드(ID 또는 링크) 공유
- 친구 B가 추천인 코드로 **앱 가입** 완료
- A·B 모두 이벤트 자동 응모

### 5.2 이벤트 상세

| 항목 | 내용 |
|------|------|
| 트리거 | 피추천인 앱 가입 완료 |
| 보상 방식 | 추첨 (건수 비례 아닌 순수 추첨) |
| 보상 종류 | 상품권 (운영 정책으로 결정) |
| 중복 참여 | 추천 건수 누적 가능 (응모권 누적) |
| 금지 | 병원 방문·상담 완료 연동 트리거 |

## 6. 병원 탐색

### 6.1 구조 — A+B 듀얼

**진입 경로**
- 하단 탭 "병원 탐색" 직접 진입 (A — 검색·필터 중심)
- 커뮤니티 비포&애프터 게시물의 병원 태그 탭 (B — 자연 유입)

### 6.2 리스트 뷰

**검색 + 필터**

| 항목 | 내용 |
|------|------|
| 검색 | 병원명, 시술 분야 키워드 |
| 필터 | 시술 분야 (카테고리·세부 시술) / 지역 |
| 정렬 | 시술 분야 일치순 / 거리 가까운 순 / 태깅 후기 많은 순 |

**병원 카드 노출 항목**

| 항목 | 내용 |
|------|------|
| 병원명 + 대표 이미지 | — |
| 전문 시술 분야 | 칩 형태 최대 3개 |
| 위치·거리 | 예: "강남구 · 1.2km" |
| 태깅 후기 수 | 예: "후기 23개" |

### 6.3 지도 뷰

| 항목 | 내용 |
|------|------|
| 전환 | 리스트 ↔ 지도 뷰 버튼 |

### 6.4 병원 상세 페이지

| 섹션 | 내용 |
|------|------|

### 7.3 면책 고지 *(v1.4 신규)*

> ⚠️ 채팅 내 의료 상담 시 비대면 진료 규제 위반 가능성 방지.

| 항목 | 내용 |
|------|------|
| 노출 시점 | 채팅방 개설 시 시스템 메시지로 자동 표시 |

| 고정 여부 | 채팅방 상단에 항상 고정 배너로 표시 |

### 7.4 채팅 종료 조건 — 3가지 모두 적용

| 조건 | 내용 |
|------|------|
| 유저 직접 종료 | "상담 종료" 버튼 → 확인 다이얼로그 → 종료 |
| 상담 완료 처리 | 유저 또는 병원이 "상담 완료" → 양측 확인 후 종료 |
| 자동 종료 | 마지막 메시지 기준 30일 비활성 시 자동 종료 (운영 정책으로 조정 가능) |

종료 후 채팅방은 읽기 전용 전환 (재개설 불가, 기록만 열람)

### 7.5 채팅 기록 보관

| 항목 | 내용 |

## 8. 회원 인증 및 계정 관리 *(v1.4 신규)*

### 8.1 일반 유저 가입 플로우

```
앱 실행 → 온보딩 (3장 스와이프) → 카카오 로그인
→ 이용약관 동의 (필수 3건 + 선택 2건)
→ 닉네임 설정 → 관심 시술 선택 (선택) → 메인화면
```

| 단계 | 상세 |
|------|------|
| 온보딩 | 3장 스와이프: ① 서비스 소개 ② 역경매 설명 ③ 커뮤니티 소개 |
| 소셜 로그인 | MVP: 카카오 로그인 단독 / MVP-B: Apple 로그인 추가 |
| 이용약관 동의 | 필수: 서비스 이용약관, 개인정보 수집·이용, 만 18세 이상 확인 |
| 선택 동의 | 마케팅 수신 동의, 민감정보(신체 사진) 수집 동의 |
| 닉네임 | 2~10자, 특수문자 제외, 중복 검사 |
| 관심 시술 | 칩 멀티셀렉 (스킵 가능), 추후 홈 피드 개인화에 활용 |

### 8.2 연령 제한 *(v1.4 신규)*

| 항목 | 내용 |
|------|------|
| 최소 연령 | 만 18세 이상 |
| 확인 방식 | MVP: 자기 신고 체크박스 ("만 18세 이상입니다") |
| 향후 | 본인인증(PASS 등) 도입 시 실제 연령 검증 전환 |
| 미성년자 접근 시 | "만 18세 미만은 서비스를 이용할 수 없습니다" + 가입 차단 |

### 8.3 알림 설정 *(v1.4 신규)*

마이페이지 > 설정 > 알림 관리

| 알림 카테고리 | 기본값 | 변경 가능 |
|-------------|--------|----------|
| 상담 도착 알림 | ON | O |
| 채팅 메시지 알림 | ON | O |
| 채팅 종료 예고 알림 | ON | O |
| 커뮤니티 댓글·좋아요 | ON | O |
| 이벤트·공지 | ON | O |
| 마케팅 알림 | OFF (동의 시 ON) | O |

### 8.4 계정 탈퇴 프로세스 *(v1.4 신규)*

> ⚠️ 신체 사진 등 민감정보 포함. 탈퇴 시 완전 삭제 처리 필수.

```
마이페이지 → 설정 → 계정 탈퇴 → 탈퇴 사유 선택 (선택)
→ 안내 문구 확인 → "탈퇴합니다" 텍스트 직접 입력 → 최종 확인 → 탈퇴 완료
```

| 항목 | 내용 |
|------|------|
| 탈퇴 유예 기간 | 7일 (유예 기간 내 로그인 시 탈퇴 철회) |
| 즉시 처리 | 로그인 차단, 프로필 비공개 전환 |
| 7일 후 처리 | 개인정보 완전 삭제 (닉네임, 사진, 상담 기록, 채팅 기록) |
| 커뮤니티 게시물 | 작성자명 "탈퇴한 사용자"로 변환, 게시물 본문은 유지 (유저 선택에 따라 삭제 가능) |
| 진행 중 상담 | 탈퇴 신청 시 진행 중 상담은 즉시 종료, 병원에 "유저가 탈퇴했습니다" 알림 |
| 재가입 | 탈퇴 완료 후 동일 카카오 계정으로 신규 가입 가능 (이전 데이터 복구 불가) |

## 12. 이미지 처리 파이프라인 *(v1.4 신규)*


| 포맷 변환 | WebP 변환 (CDN 전송 최적화) |

## 13. 기술 스택

### 13.1 단계별 인프라 전략 *(v1.4 개선)*

> ⚠️ MVP부터 10만 동접 인프라를 구축하는 것은 과잉 설계. 단계별로 확장한다.

```
Phase 1 (MVP, 0–1만 유저)          Phase 2 (1만–5만 유저)          Phase 3 (5만+ 동접)
─────────────────────────          ────────────────────────          ─────────────────────
Docker Compose + 단일 서버          Docker Swarm / 다중 서버           Kubernetes + HPA
Caddy (리버스 프록시)                Nginx (L7 LB)                    Nginx + K8s LB 이중
PostgreSQL (단일)                   PostgreSQL + PgBouncer            PostgreSQL HA + PgBouncer
Redis (단일)                        Redis Sentinel                    Redis Cluster
Go API (단일 인스턴스)               Go API (2–4 인스턴스)             Go API (HPA 자동 확장)
Centrifugo (단일)                   Centrifugo (단일)                 Centrifugo 클러스터
PostgreSQL LIKE 검색                PostgreSQL trigram                 Elasticsearch
Go goroutine 비동기                 Go goroutine 비동기               RabbitMQ + Worker
Cloudflare R2                      Cloudflare R2                     Cloudflare R2 또는 MinIO
─                                  ─                                 Terraform (IaC)
```

### 15.2 MVP-A (클로즈드 베타, 12–14주 목표)

> ⚠️ v1.3의 8~10주에서 **12~14주로 현실화**. 회원 인증, 병원 승인, 탈퇴 등 필수 기능 추가로 인한 일정 조정.

**포함**
- 회원가입/로그인 (카카오) + 온보딩 + 탈퇴
- 역경매 상담 요청 플로우 (갤러리 업로드, 카메라 가이드 없이)
- 병원 가입 승인 프로세스 + 상담 발송 + 금액 키워드 필터링
- 1:1 채팅 (Centrifugo) + 면책 고지
- 비포&애프터 게시판 (자율 커뮤니티)
